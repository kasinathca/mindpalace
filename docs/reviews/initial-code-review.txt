4. SECURITY OBSERVATIONS
────────────────────────────────────────────────────────────────

4.1 Implemented Security Controls
───────────────────────────────────

  Helmet
    Strict Content Security Policy, HSTS (1 year + subdomains),
    XSS and clickjacking protection headers.

  CORS
    Origin-locked to CORS_ORIGIN environment variable with
    credential support. No wildcard origins.

  Rate Limiting
    Three-tier strategy:
      - Global:           100 requests per 15 minutes
      - Auth endpoints:    20 requests per 15 minutes
      - Bookmark create:   30 per window per user

  JWT Authentication
    Short-lived access tokens (15 min) + HttpOnly cookie refresh
    tokens (7 days). JWT secrets validated at startup to be at
    least 32 characters.

  Password Hashing
    bcrypt with configurable rounds (default 12, max 31 enforced
    by Zod env schema).

  Input Validation
    All routes gated by Zod schemas via validate() middleware.
    No raw req.body access in services.

  SSRF Guard (archiver.ts)
    Blocks outbound requests to:
      127.x, 10.x, 172.16-31.x, 192.168.x, 169.254.x,
      AWS/GCP/Azure metadata service hostnames.

  HTML Sanitisation
    DOMPurify applied to all Readability-extracted article content
    before storing in the database.

  Password Reset Tokens
    Stored as SHA-256 hash — never plain text. One-time use
    enforced via usedAt timestamp.

  Request Logging
    Authorization header and password body field are redacted
    (shown as [REDACTED]) in all log output via pino-http.


4.2 Items to Verify / Discuss
───────────────────────────────

  Refresh Token Revocation
    Current implementation is stateless (signed JWT, not stored
    server-side). Logout-all-devices / forced revocation is noted
    in auth.controller.ts as deferred to Phase 3.
    Action: confirm this is acceptable for Phase 1 go-live.

  emailVerified Flag
    The User model has an emailVerified field but the login flow
    does not currently enforce it (any user can log in regardless).
    Action: confirm if email verification enforcement is in scope
    for Phase 1 or explicitly deferred.

  File Upload Memory Usage
    Multer uses memoryStorage for bookmark HTML imports (max 10 MB).
    Under concurrent imports this memory is not released until GC.
    Action: confirm expected concurrent usage and whether disk
    storage is preferred.

  Screenshot Storage Path
    screenshotPath is stored as a relative path in the database.
    Action: verify the read endpoint applies path validation to
    prevent directory traversal.

5. DATA MODEL SUMMARY
────────────────────────────────────────────────────────────────

  Model                Key Points
  -------------------  --------------------------------------------------------
  User                 CUID primary key, unique email, passwordHash (never plain
                       text), theme and view preferences stored inline.

  PasswordResetToken   tokenHash (SHA-256), expiresAt, usedAt for one-time use.
                       Cascades deleted when User is deleted.

  Collection           Self-referential tree. parentId -> id with CASCADE delete.
                       sortOrder field for manual ordering.

  Bookmark             searchVector (tsvector) maintained by PostgreSQL trigger
                       with GIN index. Cursor pagination on
                       (userId, createdAt DESC).

  Tag                  Unique per user, names normalised to lowercase in service
                       layer. M:N to Bookmark via BookmarkTag junction table.

  PermanentCopy        One-to-one with Bookmark. Stores rawHtml, articleContent
                       (Readability output), and failureReason if capture failed.

  Annotation           Belongs to PermanentCopy. positionData stored as JSON
                       (start/end offsets + XPath or CSS selector).

6. BACKGROUND JOB ARCHITECTURE
────────────────────────────────────────────────────────────────

All three workers run in a separate Node.js process (worker.ts)
so slow network jobs never block HTTP responses.

  Queue         Trigger                       Action
  ------------  ----------------------------  --------------------------------
  metadata      On bookmark create            Fetch URL -> extract title,
                                              description, OG image, favicon.
                                              Update bookmark record.

  archive       On bookmark create            Fetch URL -> Readability +
                                              DOMPurify -> store
                                              PermanentCopy record.

  link-health   Nightly repeatable schedule   HEAD (fallback GET) URL ->
                + on-demand check             classify as OK, BROKEN, or
                                              REDIRECTED. Update linkStatus.

7. FRONTEND TOKEN STRATEGY
────────────────────────────────────────────────────────────────

  Login response  ->  { accessToken, refreshToken, user }
                             |               |
                      Zustand authStore   HttpOnly cookie
                      (sessionStorage)   (path=/api/v1/auth)

  On every request:
    Axios request interceptor reads accessToken from authStore
    and attaches it as  Authorization: Bearer <token>

  On 401 response:
    Single token refresh attempt using refreshToken.
    If refresh succeeds  -> update tokens, retry original request.
    If refresh fails     -> call _forceLogout(), redirect to /login.


────────────────────────────────────────────────────────────────
8. KEY PATTERNS
────────────────────────────────────────────────────────────────

  Thin Controllers
    Controllers contain no business logic. They extract inputs,
    call one service method, and format the HTTP response.
    All logic lives in the corresponding *.service.ts file.

  validate() Middleware
    Wraps Zod .parse() for body/params/query. On failure it passes
    the ZodError to the global errorHandler which formats it into
    a consistent { success: false, error, issues[] } response.

  AppError
    A typed Error subclass carrying a statusCode. Thrown in
    service layer, caught by errorHandler, returned as a typed
    HTTP error response. Prevents leaking stack traces.

  Cache Pattern
    Redis cacheGet/cacheSet/cacheDel used in collections and tags
    services (5-minute TTL). All cache failures are silently
    swallowed — the cache is best-effort and never blocks the
    primary database path.

  req.user! Non-null Assertion
    Safe throughout the codebase because jwtAuthGuard always runs
    before any protected route handler. TypeScript is aware via
    the express.d.ts module augmentation.

  Cursor Pagination
    Bookmarks are paginated using a (createdAt + id) cursor rather
    than SQL OFFSET. This is consistent under concurrent writes
    and does not degrade at large page numbers.

  Full-Text Search (FTS) Strategy
    The searchVector tsvector column is written exclusively by a
    PostgreSQL trigger (not by Prisma). The search service runs a
    raw SQL query to get ranked bookmark IDs, then passes those IDs
    to a standard Prisma findMany to load typed records with
    relations. Raw SQL handles ranking; Prisma handles type safety.
